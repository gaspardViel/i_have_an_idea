Le changement de paradigme

De "copier-coller avec ChatGPT" à "Claude travaille dans ton manuscrit"

Tu as un brouillon avancé. Tu sais utiliser ChatGPT ou Claude.ai. Jusqu'ici tu travailles probablement comme ça : tu copies un passage, tu le colles dans le chat, tu demandes quelque chose, tu recolles la réponse dans ton document.

Ça marche. Mais c'est lent, et il y a un problème plus profond : Claude ne voit jamais l'ensemble du livre. Il répond à un fragment sans savoir ce qui precède ou suit.

L'idée centrale de Claude Code

Claude Code est une interface en ligne de commande (CLI) qui te permet de donner à Claude l'accès direct à tes fichiers. Au lieu de copier-coller, Claude lit et modifie tes fichiers lui-même.

L'analogie : Avec Claude.ai web, tu décris ton appartement à un architecte par téléphone. Avec Claude Code, l'architecte est dans l'appartement avec toi, il voit les murs, les mesures, la lumière — et il peut tout de suite esquisser des changements.

Comment ça marche en pratique

Voici les deux workflows comparés. Pour une tâche simple : "Révise ce paragraphe pour qu'il corresponde au ton du reste de ma mémoire."

— AVEC CLAUDE.AI WEB —
1
Ouvre Claude.ai Nouvelle conversation
2
Copie des extraits représentatifs du ton 3-4 paragraphes que tu choisis manuellement pour "montrer" le style
3
Colle le paragraphe à réviser + Explique le contexte (qui parle ? quelle époque ? quel état d'esprit ?)
4
Recolles la réponse dans ton document Si c'est dans le mauvais sens, recommence depuis le début
— AVEC CLAUDE CODE —
1
Ouvre le terminal dans ton dossier de livre Tape claude et appuie sur Entrée
2
Écris ta demande en langage naturel "Révise le paragraphe 3 du chapitre 7 pour qu'il corresponde au ton du chapitre 2"
3
Claude lit tes fichiers, modifie le passage Le fichier est édité directement. Tu approuves ou demandes d'ajuster.

Et la "bible du livre" : CLAUDE.md

Claude Code lit automatiquement un fichier appelé CLAUDE.md au démarrage. C'est là où tu mets tout ce que Claude doit savoir sur ton livre — une fois pour toutes :

Exemple de CLAUDE.md pour une mémoire :

Ce manuscrit est une autobiographie. La narratrice s'appelle Sylvie, la voix est à la première personne, passé composé, registre intime mais jamais sentimental. Les chapitres 1-4 couvrent l'enfance à Lyon (1978-1990). Le ton vire à l'humour noir dans la deuxième partie. Ne jamais utiliser le mot "ressentir".

Chaque fois que tu lances Claude Code dans ton dossier de livre, il commence avec ce contexte déjà chargé. Plus besoin de réexpliquer à chaque session.

Et VS Code dans tout ça ?

VS Code est l'éditeur où tu écris. Claude Code peut s'y intégrer, mais il peut aussi fonctionner en parallèle dans un terminal. La leçon suivante couvrira l'installation. Pour l'instant, retiens l'essentiel : VS Code = ton espace d'écriture, Claude Code = ton assistant qui vit dans le même espace.

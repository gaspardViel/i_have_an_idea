Anatomie d'un skill

Ce qu'est vraiment un skill, pourquoi ça marche hors du code,
et comment en lire un sans se perdre.

L'idée centrale

Un skill, c'est une procédure écrite une fois, exécutée par Claude autant de fois que vous voulez.

Ce n'est pas du code. Ce n'est pas un plugin. C'est un fichier texte qui dit à Claude : "quand on t'invoque pour cette tâche, voilà comment tu opères."

Pensez à un skill comme à un mode opératoire dans un centre de méditation : une fiche lamifiée sur l'armoire qui dit "Pour préparer la salle Dhamma : vérifier les coussins → nettoyer l'autel → vérifier la ventilation." Claude lit cette fiche et l'applique — chaque fois, sans oubli.

La structure d'un skill

Chaque skill vit dans un dossier .claude/skills/nom-du-skill/ et contient au minimum un fichier SKILL.md.

commission-entretien/           ← votre projet
└── .claude/
    └── skills/
        ├── compte-rendu/       ← un skill
        │   └── SKILL.md
        ├── todo-update/        ← un autre skill
        │   └── SKILL.md
        └── ordre-du-jour/
            ├── SKILL.md
            └── REFERENCE.md   ← si les instructions sont longues

Anatomie d'un SKILL.md

Un SKILL.md a deux parties : le frontmatter (métadonnées) et le corps (les instructions).

Exemple — SKILL.md du skill /compte-rendu
---
name: compte-rendu
description: Génère un compte rendu structuré depuis des notes brutes
             ou une transcription vocale. Utiliser après une réunion
             de la Commission Entretien.
---

# Compte Rendu — Commission Entretien

## Ce que tu reçois
Des notes brutes, une transcription, ou un résumé oral.

## Ce que tu produis
Un fichier Markdown avec :
- Date, participants
- Décisions prises (numérotées)
- Tâches créées (qui fait quoi, avant quand)
- Points reportés à la prochaine réunion

## Ton comportement
1. Demande la date si non précisée.
2. Identifie les tâches → ajoute-les à TODO.md.
3. Garde un ton neutre et factuel.
4. Propose un titre de fichier : YYYY-MM-DD-CR-commission.md

Les deux éléments clés du frontmatter

name

Le nom devient la commande. name: compte-rendu → vous tapez /compte-rendu. Simple.

description

C'est ce que Claude lit pour décider automatiquement de charger le skill, même si vous ne l'invoquez pas explicitement. Une bonne description = Claude qui se déclenche au bon moment sans qu'on le lui demande.

La description est le déclencheur automatique. Le name est le déclencheur manuel. Les deux sont complémentaires.

Skills code vs skills non-code — même logique

Ingénieur logiciel
/tdd — tests avant le code
/diagnose — débugger méthodiquement
/deploy — livrer en prod
Coordinateur de commission
/compte-rendu — structurer une réunion
/todo-update — mettre à jour les tâches
/ordre-du-jour — préparer la prochaine réunion

La mécanique est identique. Seul le domaine change. Claude n'a pas besoin de code pour être utile — il a besoin d'instructions claires sur comment opérer.

Ce que Claude fait quand vous invoquez un skill

  1. Il lit le SKILL.md correspondant.
  2. Il charge éventuellement REFERENCE.md si il existe.
  3. Il exécute les instructions en les adaptant au contexte courant.
  4. Il utilise les outils disponibles (lire des fichiers, écrire, appeler des agents).

Vous n'avez rien à répéter d'une session à l'autre. Le skill est votre mémoire opérationnelle.

Vérification — 3 questions

  1. Si vous créez un skill appelé bilan-semaine, quelle commande tapez-vous pour l'invoquer ?
  2. Quelle partie du SKILL.md permet à Claude de se déclencher automatiquement sans que vous tapiez une commande ?
  3. Quelle est la différence entre SKILL.md et REFERENCE.md ? Quand utilise-t-on l'un vs l'autre ?

Répondez directement dans le chat — Claude corrigera et passera à la suite.

Exercice pratique

Décrivez en 3-5 phrases ce que devrait faire votre premier skill /compte-rendu pour votre commission :

  • Qu'est-ce qu'il reçoit en entrée ? (notes ? transcription vocale ?)
  • Qu'est-ce qu'il produit ? (un fichier ? dans quel format ?)
  • Quelles sont les 2-3 règles métier propres à votre commission ?

Répondez dans le chat. On construira le skill ensemble dans la leçon 2.
